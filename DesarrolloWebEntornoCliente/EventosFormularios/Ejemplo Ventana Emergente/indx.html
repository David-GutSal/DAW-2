¿Qué es la Propagación de Eventos (Bubbling)?

En el modelo de eventos del DOM, cuando ocurre un evento en un elemento, este no solo se ejecuta en ese elemento, sino que también se propaga a sus ancestros en el árbol DOM. Esta propagación ocurre en tres fases:

  1. Fase de Capturing (Captura): El evento desciende desde el elemento raíz hasta el elemento objetivo.
  2. Fase de Target (Objetivo): El evento alcanza el elemento objetivo.
  3. Fase de Bubbling (Burbujeo): El evento asciende desde el elemento objetivo hasta el elemento raíz.

Por defecto, los event listeners se ejecutan durante la fase de bubbling (a menos que se especifique lo contrario con el tercer parámetro de addEventListener).

// Código JavaScript para este ejemplo:
// Agregar event listeners a todos los elementos
const elements = document.querySelectorAll('.element');

elements.forEach(element => {
  // Para la fase de bubbling (por defecto)
  element.addEventListener('click', function(event) {
    logEvent(event, 'BUBBLING');
  });

  // Para la fase de capturing (cuando se activa el toggle)
  element.addEventListener('click', function(event) {
    logEvent(event, 'CAPTURING');
  }, true); // true indica fase de capturing
});

// Detener la propagación cuando el botón está activado
if (stopPropagationEnabled) {
  event.stopPropagation();
  logEvent(event, 'STOPPED (no más propagación)');
}

En este ejemplo, al hacer clic en el elemento más interno (div4), puedes observar cómo el evento se propaga hacia arriba a través de todos sus ancestros (div3 → div2 → div1).
